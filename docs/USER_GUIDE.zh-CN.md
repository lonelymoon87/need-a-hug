# need-a-hug 使用手册

## 适合什么时候用

当你在和 Agent 工作时，发现自己已经不只是“任务卡住”，而是开始难受、自责、焦虑、崩溃，可以用它。

典型场景：

- bug 修不出来，开始怀疑自己
- AI 一直错，你越来越烦
- 做了很久的项目，发现别人已经做得更好
- 拖延太久，觉得自己错失机会
- 工作/项目把你压得喘不过气
- 只是想被安慰一下

## 怎么触发

直接说：

```text
/hug
/need-a-hug
抱抱我
安慰我一下
我撑不住了
我想哭
我好累
```

在 Codex 里也可以直接引用：

```text
$need-a-hug
```

## 它会怎么回应

首轮可能会像这样：

```text
🫂 先抱抱你。

先不急着处理问题。看起来这件事已经把你耗得很厉害了，可能还让你开始怀疑自己是不是不够好。先别急着信这个。

你可以先什么都不做。喝口水，坐一会儿，慢慢缓一下。

我在。我们先不急着做决定。等你缓一点，如果你愿意，我们再一起看下一步。
```

后续不会每轮都重复 `🫂 先抱抱你。`，而是接着你的话继续回应。

## 初始化偏好

如果你希望它更贴近你，可以说：

```text
/hug:init
```

它会问：

```text
以后我可以怎么称呼你？

不想说也没关系，我们直接继续。
```

它不应该从系统用户名、文件路径、Git 配置、邮箱里猜你的名字。
它也不应该从语言、时区、路径、账号信息里猜你的国家或地区。
语气和偏好不做问卷。Agent 应该从后续对话里慢慢感知，而不是一开始就让你填表。

## 怎么退出

说：

```text
/hug:off
/back-to-work
回到任务
继续做事
别安慰了，直接解决问题
```

## 它不会做什么

它不会：

- 诊断你
- 自称心理咨询师
- 替代治疗或医疗建议
- 让你依赖 AI
- 每轮重复模板
- 强迫你选“安慰还是建议”
- 用“加油”“你可以的”糊弄过去

## 危机情况

如果你提到自伤、自杀、即时危险、虐待或医疗紧急情况，Agent 会优先建议你联系现实世界里的帮助：当地紧急服务、身边可信任的人，或与你明确提供的国家/地区匹配的危机帮助资源。

这不是因为它不想陪你，而是因为那种时刻，真实的人和即时安全更重要。

如果你只是长期疲惫、睡不好、很难受，但没有自伤或即时危险，Agent 不应该突然给你危机热线。它可以温和地建议你在失眠持续、白天明显受影响时，考虑找医生或心理专业人士聊聊。
